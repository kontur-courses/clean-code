# Спецификация языка разметки

Посмотрите этот файл в сыром виде. Сравните с тем, что показывает github.

Процессору на вход подается одна строка — параграф текста. 
На выходе должен быть HTML-код этого параграфа.

Параграф - часть текста,
в которой не встречаются два переноса строки подряд

Текст _окруженный с двух сторон_  одинарными символами подчерка 
должен помещаться в HTML-тег em вот так:

`Текст <em>окруженный с двух сторон</em>  одинарными символами подчерка 
должен помещаться в HTML-тег em вот так:`

Любой символ можно экранировать, чтобы он не считался частью разметки. 
\_Вот это\_, не должно выделиться тегом \<em\>.

__Двумя символами__ — должен становиться жирным с помощью тега \<strong\>.

Внутри __двойного выделения _одинарное_ тоже__ работает.

Но не наоборот — внутри _одинарного \_\_двойное\_\_ не работает_.

Подчерки внутри текста c цифрами_12_3 не считаются выделением
и должны оставаться символами подчерка.

__непарные _символы не считаются выделением.

За подчерками, начинающими выделение, должен следовать непробельный символ.
Иначе эти_ подчерки_ не считаются выделением 
и остаются просто символами подчерка.

Подчерки, заканчивающие выделение, должны следовать за непробельным символом.
Иначе эти _подчерки _не считаются окончанием выделения 
и остаются просто символами подчерка.

В случае, __если \_одинарный тэг открыт внутри двойного__,
но закрыт вне_, он не считается выделением

Если открыто _два \_одинаковых тэга, но закрыт один_,
то выделаяется первый из них

_Одинаковые \_тэги не могут быть вложены друг_ в друга_,
при этом работает первый открытый и первый закрытый

Строка, начинающаяся с сивола #, повторенного от одного до шести раз
и заканчивающаяся переносом строки,
выделяется с помощью тэга \<h1>, \<h2>, ..., \<h6> соответственно
и является заголовком с разной силой выделения
